How It Works

Squeeze Evolve runs an evolutionary loop over candidate solutions. Each loop scores, groups, and recombines candidates, routing each group to the model best suited for its difficulty. Every stage in the pipeline is a pluggable operator you can customize.

Loop 0: Initialization

The most expensive model generates N candidate solutions per problem. Initialization quality is the strongest predictor of final accuracy, so this step always uses the best model available.

Loops 1..T: Evolution

1. Score CUSTOMIZABLE

A fitness function computes a signal for each candidate. Built-in options:

  • Group Confidence (GC) — mean of per-token confidence from log-probabilities
  • Diversity — count of unique final answers in a group

Register your own via @fitness.register("name"). See Extending Operators.

2. Select CUSTOMIZABLE

A selection function groups candidates into subsets of size K. Built-in options:

  • Uniform — random sampling without replacement
  • Weighted — fitness-weighted sampling with configurable temperature

Register your own via @selection.register("name").

3. Route

Each group's fitness determines which model handles it. Thresholds are computed per-problem from configurable percentiles.

FitnessAssignment
Low (hard)Most expensive model
MediumMid-tier model(s)
High (easy)Cheapest model
Full consensusLite aggregation (no LLM call)

Supports any number of model tiers. See Configuration for details on N-model routing.

4. Recombine CUSTOMIZABLE

A recombination function builds the prompt sent to the assigned model. Each model receives its groups in parallel and generates one refined candidate per group. All tiers execute concurrently.

Built-in: aggregate (task-aware), synthesize (minimal). Register your own via @recombination.register("name").

5. Update CUSTOMIZABLE

An update function decides how new candidates merge with the population.

  • Replace — new candidates become the entire population
  • Accumulate — new candidates are appended, growing the population

Register your own via @update.register("name").
