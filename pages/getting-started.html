Getting Started

Install Squeeze Evolve, serve your models, run your first evolutionary loop, and understand the output.

Install

Requirements: Python 3.10+, uv (recommended) or pip, Git.

git clone --recurse-submodules git@github.com:Nietzsche2000/squeeze-evolve.git
cd squeeze-evolve

# Recommended
uv sync --dev

# Or with pip
pip install -e ".[dev]"

Optional: forked vLLM

The external/vllm/ submodule contains a forked vLLM with a custom prefill path that computes confidence scores directly on GPU (4-10x lower scoring latency).

VLLM_USE_PRECOMPILED=1 uv pip install --editable external/vllm

Optional: cloud storage

# Amazon S3
uv sync --dev --extra aws

# Google Cloud Storage
uv sync --dev --extra gcs

Verify the install:

squeeze-evolve-client --help

Serve your models

Squeeze Evolve talks to models over the OpenAI-compatible API. You need at least one running server. Here we set up two tiers with vLLM:

# Cheap model (model_0)
vllm serve Qwen/Qwen3-30B-A3B-Instruct-2507 \
  --port 8001 \
  --tensor-parallel-size 4 \
  --enable-prefix-caching

# Expensive model (model_1)
vllm serve Qwen/Qwen3-235B-A22B-Instruct-2507 \
  --port 8000 \
  --tensor-parallel-size 8 \
  --enable-prefix-caching

Verify both are healthy:

curl http://localhost:8001/v1/models
curl http://localhost:8000/v1/models
Tip You can use any OpenAI-compatible server: vLLM, SGLang, TGI, or cloud APIs like OpenAI/Together/OpenRouter. Just update the base_url and api_key in your config.

Write a config

Create my_run.yaml:

run_name: my_first_run

routing:
  k: 4              # group size
  population: 16    # candidates per problem in loop 0
  loops: 5          # total iterations
  confidence_percentiles: [50.0]
  fitness: confidence
  selection: uniform
  update: replace
  recombination: aggregate
  task: math
  seed: 1234

models:
  - name: Qwen/Qwen3-30B-A3B-Instruct-2507
    base_url: http://localhost:8001/v1
    max_tokens: 8192
    temperature: 0.7
    top_p: 0.8

  - name: Qwen/Qwen3-235B-A22B-Instruct-2507
    base_url: http://localhost:8000/v1
    max_tokens: 8192
    temperature: 0.7
    top_p: 0.8

scoring_model:
  name: Qwen/Qwen3-30B-A3B-Instruct-2507
  base_url: http://localhost:8001/v1
  prompt_logprobs: true
  vllm_extensions: true

checkpoint_dir: ./artifacts/checkpoints
metrics_path: ./artifacts/metrics.json

What this does:

  • Loop 0: The 235B model generates 16 candidates per problem
  • Loops 1-4: Candidates are grouped into subsets of 4, scored by confidence, and routed: low-confidence groups go to 235B, high-confidence groups go to 30B
  • The 50th percentile threshold means roughly half the groups go to each model per problem

Run it

# Run on 5 problems, print results to stdout
squeeze-evolve-client \
  --config my_run.yaml \
  --input data/aime25/test.parquet \
  --n-problems 5

# Or save to a file
squeeze-evolve-client \
  --config my_run.yaml \
  --input data/aime25/test.parquet \
  --n-problems 5 \
  --output results.json

Understand the output

The output JSON has three top-level keys:

{
  "run_id": "my_first_run_a3f7b2c1",
  "metrics": [ ... ],
  "problems": [ ... ]
}

metrics

One entry per loop:

FieldMeaning
loopLoop index (0 = init)
model_0_countGroups sent to cheap model
model_1_countGroups sent to expensive model
lite_countGroups handled by lite agg
total_input_tokensTotal prompt tokens across all models
total_output_tokensTotal completion tokens
time_scoring_sTime spent on confidence scoring
time_generation_sTime spent on model generation
mean_confidenceAverage confidence across all candidates
eval_mean_accAccuracy (if evaluation is enabled)

problems

Final state of each problem after all loops:

{
  "orig_prompt": "Find the sum of all positive integers n...",
  "gt": "42",
  "candidates": [
    "The answer is 42. Here's why...",
    "After careful analysis, we get 42...",
    "..."
  ],
  "routing_details": {
    "routes": ["model_0", "model_1", "model_0", ...],
    "thresholds": [0.847]
  }
}

Checkpoints

A checkpoint is saved after every loop at checkpoint_dir:

ls artifacts/checkpoints/
# my_first_run_loop0.json
# my_first_run_loop1.json
# ...
# my_first_run_loop4.json

To resume a crashed run, set resume: true in your config.

Adding evaluation

To see accuracy during the run:

routing:
  evaluation: exact_match
  task: math

This compares candidates against gt (ground truth) and reports eval_mean_acc, eval_pass_at_k, and eval_majority_vote in the metrics.

Scaling up

More models

Add a third model and adjust percentiles:

routing:
  confidence_percentiles: [33.0, 66.0]  # 2 thresholds for 3 tiers

models:
  - name: Qwen/Qwen3-30B-A3B-Instruct-2507     # cheap
    base_url: http://localhost:8002/v1
  - name: Qwen/Qwen3-235B-A22B-Instruct-2507    # mid
    base_url: http://localhost:8001/v1
  - name: openai/gpt-5-mini                      # expensive
    base_url: https://api.openai.com/v1
    api_key: sk-...

Lite aggregation

Skip the LLM entirely for high-consensus groups:

routing:
  lite_fraction: 0.25    # top 25% of easy groups use majority vote

Weighted selection + accumulation

Focus on high-fitness candidates and grow the population over time:

routing:
  selection: weighted
  selection_temperature: 0.5   # lower = more exploitation
  update: accumulate           # append new candidates instead of replacing
  groups: 32                   # more groups than population = oversampling
