Design

  • Async-first. All model tiers execute in parallel via asyncio.gather().
  • No vLLM dependency. Communicates through the OpenAI chat/completions API only.
  • Pydantic config. Type-safe parsing with automatic validation.
  • Registry pattern. All operators are plain functions registered by name.
  • Pluggable storage. Auto-detected from path prefix.

Data flow

CLI / Server
  → load_run_config(yaml) → RunConfig
  → load_dataset(parquet) → [{orig_prompt, gt}]
  → RoutingOrchestrator(RunConfig)
      ├─ Loop 0: backends[-1].generate_batched()
      └─ Loop t:
          ├─ score_population() → candidate_scores
          ├─ selection() → groups
          ├─ fitness() → group fitnesses
          ├─ compute_thresholds() + assign_routes()
          ├─ asyncio.gather(tier_0, tier_1, ..., lite)
          ├─ merge → update population
          └─ save checkpoint + emit metrics
