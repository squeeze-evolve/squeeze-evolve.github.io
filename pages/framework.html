Using the Framework

How the operator system works, how to register custom operators, and how to build complete custom pipelines.

The operator lifecycle

When the orchestrator starts, it reads your config and resolves every operator name to a function:

# Inside RoutingOrchestrator.__init__()
self._fitness_fn   = fitness.get(cfg.routing.fitness)        # "confidence" → group_confidence()
self._select_fn    = selection.get(cfg.routing.selection)     # "uniform"    → select_uniform()
self._update_fn    = update.get(cfg.routing.update)           # "replace"    → update_replace()
self._lite_agg_fn  = lite_agg.get(cfg.routing.lite_method)    # "majority"   → lite_aggregate_majority()
self._recomb_fn    = recombination.get(cfg.routing.recombination)  # "aggregate" → aggregate_prompt()
self._eval_fn      = evaluation.get(cfg.routing.evaluation)   # "none"       → eval_none()

Then during each evolution loop, the orchestrator calls them in this order:

for each problem:
    1. _select_fn(candidates, k, groups, scores=...) → groups, indices
    2. _fitness_fn(scores_per_group)                  → group fitness (one float per group)
    3. compute_thresholds(fitnesses, percentiles)     → thresholds
    4. assign_routes(fitnesses, thresholds, n_models)  → route per group
    5. _recomb_fn(query, group_candidates)            → prompt string (sent to assigned model)
    ... model generates responses ...
    6. _update_fn(old_population, new_candidates)      → next population
    7. _eval_fn(candidates, ground_truth)              → metrics dict

Every one of these steps is a registered operator you can replace.

How registration works

Each operator family has a global Registry instance defined in operators.py:

from squeeze_evolve.core.registry import Registry

fitness:       Registry[Callable] = Registry("fitness")
selection:     Registry[Callable] = Registry("selection")
lite_agg:      Registry[Callable] = Registry("lite_agg")
update:        Registry[Callable] = Registry("update")
recombination: Registry[Callable] = Registry("recombination")
evaluation:    Registry[Callable] = Registry("evaluation")
configs:       Registry[Callable] = Registry("configs")

Built-in operators register themselves at import time:

@fitness.register("confidence")
def group_confidence(scores):
    return float(np.mean(scores))

Your custom operators work the same way. Import the registry, decorate your function, and the name you pass to register() becomes the string you use in YAML.

Writing a custom fitness function

A fitness function takes per-candidate scores for a group and returns a single float. Higher means easier/more confident.

import numpy as np
from squeeze_evolve import fitness

@fitness.register("min_confidence")
def min_confidence(scores, **kwargs):
    """Use the worst candidate's confidence as the group signal.
    Groups where even one candidate is uncertain get the expensive model."""
    return float(np.min(scores))
routing:
  fitness: min_confidence
Always accept **kwargs The orchestrator passes context like task and temperature through kwargs. Your function must accept them even if it doesn't use them.

Writing a custom selection function

A selection function receives the full candidate list, group size k, group count m, and keyword args including scores. It returns two parallel lists: string groups and index groups.

import random
from squeeze_evolve import selection

@selection.register("tournament")
def tournament_select(candidates, k, m, **kwargs):
    """Tournament selection: sample 2k candidates, keep the top-k by fitness."""
    scores = kwargs["scores"]
    groups, indices = [], []
    for _ in range(m):
        pool = random.sample(range(len(candidates)), min(k * 2, len(candidates)))
        winners = sorted(pool, key=lambda i: scores[i], reverse=True)[:k]
        indices.append(winners)
        groups.append([candidates[i] for i in winners])
    return groups, indices
routing:
  selection: tournament

Writing a custom evaluation function

An evaluation function receives the candidate list and ground truth, and returns a dict of metrics. Numeric values are automatically averaged across problems and appear as eval_<key> in the output.

import re
from collections import Counter
from squeeze_evolve import evaluation

@evaluation.register("math_boxed")
def math_boxed_eval(candidates, gt, **kwargs):
    """Extract \\boxed{} answers and compare to ground truth."""
    def extract_boxed(text):
        match = re.findall(r'\\boxed\{([^}]*)\}', text)
        return match[-1].strip() if match else ""

    answers = [extract_boxed(c) for c in candidates]
    correct = [a == str(gt).strip() for a in answers]
    majority = Counter(answers).most_common(1)[0][0] if answers else ""

    return {
        "mean_acc": sum(correct) / max(len(correct), 1),
        "pass_at_k": float(any(correct)),
        "majority_vote": float(majority == str(gt).strip()),
    }
routing:
  evaluation: math_boxed

Writing a custom recombination prompt

A recombination function builds the prompt that gets sent to the assigned model. It receives the original query and the group's candidates.

from squeeze_evolve import recombination

@recombination.register("debate")
def debate_prompt(query, candidates, **kwargs):
    """Frame recombination as a debate between candidates."""
    if not candidates:
        return query

    parts = [f"Problem: {query}\n"]
    parts.append("The following solutions have been proposed:\n")
    for i, c in enumerate(candidates, 1):
        parts.append(f"Debater {i}: {c.strip()}\n")
    parts.append(
        "You are the judge. Evaluate each debater's reasoning, "
        "identify errors, and produce the correct final answer. "
        "End with the result in \\boxed{}."
    )
    return "\n".join(parts)
routing:
  recombination: debate

Writing a custom update function

An update function decides how new candidates merge with the existing population.

from squeeze_evolve import update

@update.register("keep_unique")
def keep_unique(old, new, **kwargs):
    """Accumulate but deduplicate."""
    seen = set()
    result = []
    for c in old + new:
        key = c.strip()
        if key not in seen:
            seen.add(key)
            result.append(c)
    return result

Combining custom operators

Put everything in a single Python file:

# custom_ops.py
import numpy as np
import re
import random
from collections import Counter
from squeeze_evolve import fitness, selection, evaluation, recombination

@fitness.register("min_confidence")
def min_confidence(scores, **kwargs):
    return float(np.min(scores))

@selection.register("tournament")
def tournament_select(candidates, k, m, **kwargs):
    scores = kwargs["scores"]
    groups, indices = [], []
    for _ in range(m):
        pool = random.sample(range(len(candidates)), min(k * 2, len(candidates)))
        winners = sorted(pool, key=lambda i: scores[i], reverse=True)[:k]
        indices.append(winners)
        groups.append([candidates[i] for i in winners])
    return groups, indices

@evaluation.register("math_boxed")
def math_boxed_eval(candidates, gt, **kwargs):
    def extract(text):
        m = re.findall(r'\\boxed\{([^}]*)\}', text)
        return m[-1].strip() if m else ""
    answers = [extract(c) for c in candidates]
    correct = [a == str(gt).strip() for a in answers]
    majority = Counter(answers).most_common(1)[0][0] if answers else ""
    return {
        "mean_acc": sum(correct) / max(len(correct), 1),
        "pass_at_k": float(any(correct)),
        "majority_vote": float(majority == str(gt).strip()),
    }

@recombination.register("debate")
def debate_prompt(query, candidates, **kwargs):
    if not candidates:
        return query
    parts = [f"Problem: {query}\n"]
    for i, c in enumerate(candidates, 1):
        parts.append(f"Debater {i}: {c.strip()}\n")
    parts.append("Judge: produce the correct answer in \\boxed{}.")
    return "\n".join(parts)

Then use them together:

routing:
  fitness: min_confidence
  selection: tournament
  evaluation: math_boxed
  recombination: debate
  update: replace
  task: math
squeeze-evolve-client \
  --config my_run.yaml \
  --include-path ./ \
  --input data/aime25/test.parquet

The --include-path ./ flag tells the CLI to execute all .py files in the current directory at startup, which triggers all the @register decorators before the orchestrator resolves config strings.

Python config presets

Instead of YAML, you can define configs as Python functions and register them:

# presets/fast_math.py
from squeeze_evolve import configs
from squeeze_evolve.core.config import RunConfig, RoutingConfig, ModelConfig

@configs.register("fast_math")
def fast_math():
    return RunConfig(
        run_name="fast_math",
        routing=RoutingConfig(
            k=2, population=8, loops=3,
            fitness="confidence", selection="uniform",
            update="replace", recombination="aggregate",
            task="math",
        ),
        models=[
            ModelConfig(name="cheap", base_url="http://localhost:8001/v1",
                        temperature=0.7, top_p=0.8),
            ModelConfig(name="expensive", base_url="http://localhost:8000/v1",
                        temperature=0.7, top_p=0.8),
        ],
        scoring_model=ModelConfig(
            name="cheap", base_url="http://localhost:8001/v1",
            prompt_logprobs=True, vllm_extensions=True,
        ),
    )
squeeze-evolve-client --config fast_math --include-path presets/ --input data/aime25/test.parquet

Using the orchestrator directly

For programmatic use, skip the CLI and call the orchestrator from Python:

import asyncio
from squeeze_evolve.algorithm.orchestrator import RoutingOrchestrator, load_run_config
from squeeze_evolve.core.data import load_dataset

cfg = load_run_config("my_run.yaml")
problems = load_dataset("data/aime25/test.parquet", n_problems=5)

def on_loop(metrics):
    print(f"Loop {metrics.loop}: {metrics.confidence.mean_confidence:.4f} mean conf")

result = asyncio.run(RoutingOrchestrator(cfg).run(problems, on_loop_complete=on_loop))
print(f"Done. {len(result['metrics'])} loops completed.")

The on_loop_complete callback fires after each loop with a LoopMetrics object, useful for live monitoring or early stopping logic.
