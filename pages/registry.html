Registry Pattern

All operators are managed through a generic Registry[T] class.

API

MethodDescription
register(key)Decorator that registers a function under key
get(key)Returns the registered function, or raises KeyError
keys()Lists all registered names
key in registryChecks if a key exists

How it works

from squeeze_evolve.core.registry import Registry

# Each operator family gets its own instance
fitness: Registry[Callable] = Registry("fitness")

# Operators self-register at import time
@fitness.register("confidence")
def group_confidence(scores):
    return float(np.mean(scores))

The orchestrator resolves config strings at init:

self._fitness_fn = fitness.get(cfg.routing.fitness)
# "confidence" → group_confidence

Unknown keys produce a clear error:

KeyError: Unknown fitness operator: 'foo'. Available: confidence, diversity

Design rationale

  • No metaclass magic. Explicit decorator registration.
  • No auto-discovery. Modules must be imported (--include-path provides opt-in discovery).
  • Duplicate keys raise immediately.
